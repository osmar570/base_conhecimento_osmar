# Gestão e Estratégia de Produto

A gestão de produto foca em descobrir o que deve ser construído para entregar o máximo de valor ao usuário e ao negócio com o menor esforço possível.

---

## 🔍 Product Discovery (Descoberta de Produto)

O Discovery é o processo de reduzir a incerteza antes de começar a codificar. O objetivo é validar quatro riscos principais:
1.  **Valor:** O usuário quer/precisa disso?
2.  **Usabilidade:** O usuário consegue entender como usar?
3.  **Viabilidade:** Temos tecnologia e tempo para construir?
4.  **Viabilidade de Negócio:** Isso faz sentido para a nossa estratégia/finanças?

---

## 🔝 Frameworks de Priorização

Priorizar é a arte de dizer "não" ou "agora não" para a maioria das ideias.

### 1. RICE (Reach, Impact, Confidence, Effort)
Uma pontuação objetiva para comparar iniciativas:
- **Reach (Alcance):** Quantas pessoas isso afetará em um período?
- **Impact (Impacto):** Quanto isso ajudará o usuário (escala de 0.25 a 3)?
- **Confidence (Confiança):** Quão seguros estamos dos nossos dados (em %)?
- **Effort (Esforço):** Quanto tempo levará (em meses-homem)?
- **Fórmula:** `(R * I * C) / E`

### 2. MoSCoW
Ideal para definir o escopo de uma entrega (MVP ou Sprint):
- **Must Have:** Essencial, sem isso não há entrega.
- **Should Have:** Importante, mas não vital.
- **Could Have:** Desejável se sobrar tempo (o "perfume").
- **Won't Have:** Não será feito desta vez.

---

## 🗺️ User Story Mapping

Uma técnica visual para organizar o backlog e planejar releases focadas na jornada do usuário.
- **Espinha Dorsal (Backbone):** As atividades principais do usuário (ex: "Fazer Login", "Escolher Produto", "Pagar").
- **Histórias de Usuário:** Detalhes de cada atividade organizados verticalmente por prioridade.
- **Fatias (Slices):** Linhas horizontais que definem o escopo de cada **Release** ou **MVP**.

---

## 📝 Histórias de Usuário (User Stories)
O formato padrão para descrever requisitos de forma centrada no valor:
> **Como** [papel/usuário], **eu quero** [ação], **para que** [resultado/valor].

**Critérios de Aceite:** O "contrato" que define quando a história está realmente pronta.
