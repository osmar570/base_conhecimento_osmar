# Engenheiro de Dados

O Engenheiro de Dados é responsável por construir e manter as "pipelines" que permitem que os dados fluam de sistemas transacionais para ambientes analíticos.

## Conhecimentos Essenciais
* **Linguagens**: Python, SQL, Scala/Java.
* **Big Data**: Spark, Hadoop, Kafka.
* **Data Warehousing**: Snowflake, BigQuery, Redshift.
* **Orquestração**: Airflow, Dagster, Prefect.
* **Modelagem de Dados**: Star Schema, Snowflake Schema, Data Vault.

## Trilha de Aprendizado
1. Fundamentos de SQL e Python.
2. Cloud Computing (AWS/GCP/Azure).
3. Processamento distribuído com Spark.
4. Ferramentas de ETL/ELT e Orquestração.

## 📚 Recursos Recomendados

### Livros Fundamentais
*   **"Designing Data-Intensive Applications"** (Martin Kleppmann): A "bíblia" moderna dos sistemas de dados.
*   **"The Data Warehouse Toolkit"** (Ralph Kimball): O padrão para modelagem dimensional.
*   **"Fundamentals of Data Engineering"** (Joe Reis): Visão holística do ciclo de vida de dados.

### Cursos e Prática
*   **DataTalks.Club**: Comunidade e cursos (Data Engineering Zoomcamp).
*   **Hadoop & Spark Hub**: Documentação oficial da Apache Foundation.
*   **dbt (Data Build Tool) Documentation**: Essencial para transformações de dados modernas.
